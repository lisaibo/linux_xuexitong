一、文件解释：
myhead.h：是项目的头文件
menu.c：里面主要是针对菜单的操作
	1.showMenu_1()：用于显示菜单提示用户，并获取用户的选项
	2.menu_1()：调用showMenu_1()函数获得用户选项，使用switch语句针对不同的选项调用不同的操作函数
operate.c：文件中针对菜单的操作，编写了一系列函数
	1.newFile_1()：创建新文件,可以指定文件名，也可以包括路径（可以在当前目录下新建也可以在指定的目录下新建），以"a"打开或者新建一个文件，原有的文件不会删除，这样就可以操作现有的文件
	2.writeFile_1()：写文件，将特定内容写入第一步指定的文件
	3.readFile_1()：读文件，读第一步指定的文件全部内容
	4.changeMode_1()：修改文件权限，修改第一步指定的文件的权限，通过chmod()函数就可以实现
	5.lookMode_1()：查看文件权限，使用stat()函数获取文件详细信息
main.c：调用menu_1()菜单函数，实现程序功能

将menu.c文件生成静态库文件libstlib.a，将operate.c文件编译生成动态库文件libdylib.so

使用静态库和动态库，和main.c文件生成test可执行文件

二、主要命令：
ar -rc libstlib.a menu.o
gcc -shared -fPIC -o libdylib.so operate.c
gcc -o test main.c ./libdylib.so -L. -lstlib


三、运行结果：

********************
0.退出
1.创建新文件/录入已有文件
2.写文件
3.读文件
4.修改文件权限
5.查看当前文件的权限
********************
请输入选项(填入0~5):

说明：如果要操作文件，必须先通过选项1，创建新文件或者录入已有文件，这样后面的操作就可以对这个文件进行操作了，如果直接进行操作，会提示“请先创建新文件/录入已有文件，再执行操作！”
